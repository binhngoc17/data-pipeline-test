# data-pipeline Constitution

## Core Principles

### I. Specification-First
Every feature starts with a spec before code

### II. Test-First
Tests written before implementation; TDD enforced

### III. Code Quality
Consistent formatting, linting, and review standards

### IV. Simplicity
Start simple; avoid premature abstraction (YAGNI)

### V. Observability
Structured logging and metrics for debuggability
